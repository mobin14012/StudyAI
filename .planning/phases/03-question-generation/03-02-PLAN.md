# Plan 03-02: Backend API + Integration

## Goal

Create REST API endpoints for question generation and management, integrate with existing material service for cascade delete, and register routes in the main app.

## Prerequisites

- Plan 03-01 complete (Question model, services, schemas exist)
- Server builds successfully

## Tasks

### Task 1: Create Question Routes

**File:** `server/src/routes/question.routes.ts`

Create all question REST endpoints:

| Method | Path | Handler | Middleware |
|--------|------|---------|------------|
| POST | `/generate` | generateQuestions | authenticate, aiLimiter, validate |
| GET | `/` | listQuestions | authenticate, validate |
| GET | `/:id` | getQuestion | authenticate, validate |
| GET | `/:id/full` | getQuestionWithAnswer | authenticate, validate |
| DELETE | `/:id` | deleteQuestion | authenticate, validate |
| DELETE | `/batch` | deleteQuestionsBatch | authenticate, validate |

**Handler implementations:**

1. **generateQuestions:**
   - Validate request body with generateQuestionsSchema
   - Get user from req.user
   - Call questionService.generateOrGetCachedQuestions()
   - Return questions without answers (strip correctAnswer, explanation)
   - Return { questions, generated, cached, totalReturned }

2. **listQuestions:**
   - Parse query params with questionListQuerySchema
   - Query Question model with filters (userId, materialId, topic, difficulty, type)
   - Paginate results
   - Return questions without answers

3. **getQuestion:**
   - Find question by ID and userId
   - Return question without answer

4. **getQuestionWithAnswer:**
   - Find question by ID and userId
   - Return full question including correctAnswer and explanation

5. **deleteQuestion:**
   - Find and delete question by ID and userId
   - Return 204 No Content

6. **deleteQuestionsBatch:**
   - Validate IDs array with batchDeleteSchema
   - Delete all matching questions for userId
   - Return { deleted: count }

**Acceptance criteria:**
- [ ] All endpoints protected with authenticate middleware
- [ ] /generate uses aiLimiter middleware
- [ ] Proper error handling with AppError
- [ ] List/single question endpoints don't expose answers
- [ ] /full endpoint exposes answers (for review mode)

## API Contract Summary

### POST /api/questions/generate

**Request:**
```json
{
  "materialId": "65f...",
  "topic": "Photosynthesis",
  "difficulty": "medium",
  "types": ["mcq", "short_answer"],
  "count": 5
}
```

**Response (201):**
```json
{
  "success": true,
  "data": {
    "questions": [
      {
        "id": "65f...",
        "type": "mcq",
        "difficulty": "medium",
        "text": "What is the primary function...",
        "options": ["A", "B", "C", "D"],
        "topic": "Photosynthesis",
        "createdAt": "2026-03-18T..."
      }
    ],
    "generated": 3,
    "cached": 2,
    "totalReturned": 5
  }
}
```

### GET /api/questions

**Query params:** materialId?, topic?, difficulty?, type?, page?, limit?

**Response (200):**
```json
{
  "success": true,
  "data": [...],
  "pagination": { "page": 1, "limit": 20, "total": 45, "totalPages": 3 }
}
```

### GET /api/questions/:id/full

**Response (200):**
```json
{
  "success": true,
  "data": {
    "id": "65f...",
    "type": "mcq",
    "text": "What is...",
    "options": ["A", "B", "C", "D"],
    "correctAnswer": "A",
    "explanation": "Because...",
    "topic": "Photosynthesis",
    "difficulty": "medium"
  }
}
```
