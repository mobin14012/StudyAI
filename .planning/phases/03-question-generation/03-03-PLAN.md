# Plan 03-03: Frontend Generation UI

## Goal

Build the complete frontend UI for question generation, including the generation form, question display components, and pages. Users will be able to select a material, choose a topic, configure generation options, and view generated questions.

## Prerequisites

- Plan 03-02 complete (API endpoints working)
- Client builds successfully

## Tasks

### Task 1: Add Question Types

**File:** `client/src/types/index.ts`

Add TypeScript types for questions:

```typescript
export type QuestionType = "mcq" | "short_answer" | "true_false";
export type DifficultyLevel = "easy" | "medium" | "hard";

export interface Question {
  id: string;
  type: QuestionType;
  difficulty: DifficultyLevel;
  text: string;
  options?: string[];
  topic: string;
  materialId: string;
  createdAt: string;
}

export interface QuestionWithAnswer extends Question {
  correctAnswer: string;
  explanation: string;
}

export interface GenerateQuestionsRequest {
  materialId: string;
  topic: string;
  difficulty?: DifficultyLevel;
  types: QuestionType[];
  count?: number;
}

export interface GenerateQuestionsResponse {
  questions: Question[];
  generated: number;
  cached: number;
  totalReturned: number;
}
```

**Acceptance criteria:**
- [ ] All types exported
- [ ] Types match backend API contract

### Task 2: Create Question API Functions

**File:** `client/src/api/questions.ts`

Create API functions:
- `generateQuestionsApi(params)` - POST /api/questions/generate (60s timeout)
- `getQuestionsApi(filters)` - GET /api/questions with query params
- `getQuestionApi(id)` - GET /api/questions/:id
- `getQuestionWithAnswerApi(id)` - GET /api/questions/:id/full
- `deleteQuestionApi(id)` - DELETE /api/questions/:id
- `deleteQuestionsBatchApi(ids)` - DELETE /api/questions/batch

**Acceptance criteria:**
- [ ] All functions use api client from ./client
- [ ] Proper error handling
- [ ] 60s timeout for generation (AI takes time)

### Task 3: Create TanStack Query Hooks

**File:** `client/src/hooks/use-questions.ts`

Create React Query hooks:
- `useQuestions(params)` - list questions with filters
- `useQuestion(id)` - single question without answer
- `useQuestionWithAnswer(id)` - single question with answer
- `useGenerateQuestions()` - mutation for generation
- `useDeleteQuestion()` - mutation for delete
- `useDeleteQuestionsBatch()` - mutation for batch delete

**Acceptance criteria:**
- [ ] Proper query keys for caching
- [ ] Mutations invalidate questions query
- [ ] `enabled: !!id` for conditional queries

### Task 4: Create Question Components

**Directory:** `client/src/components/questions/`

Create these components:

#### 4a. QuestionTypeSelector.tsx
- Checkbox group for MCQ, Short Answer, True/False
- Props: `selected: QuestionType[]`, `onChange: (types: QuestionType[]) => void`
- At least one type must be selected

#### 4b. DifficultySelector.tsx
- Radio button group for Easy, Medium, Hard
- Props: `value: DifficultyLevel`, `onChange: (level: DifficultyLevel) => void`
- Default based on user level (from useAuth)

#### 4c. GenerationProgress.tsx
- Loading spinner with message "Generating questions..."
- Shows cached count if available
- Props: `isLoading: boolean`, `cached?: number`

#### 4d. QuestionCard.tsx
- Displays single question
- Shows: type badge, difficulty badge, question text
- For MCQ: shows options as list
- Expandable: click to reveal answer + explanation
- Delete button with confirmation
- Props: `question: Question | QuestionWithAnswer`

#### 4e. QuestionList.tsx
- Grid/list of QuestionCard components
- Props: `questions: Question[]`
- Empty state: "No questions generated yet"

#### 4f. GenerateQuestionsForm.tsx
- Material dropdown (from useMaterials hook - ready materials only)
- Topic badges (from selected material's topics)
- QuestionTypeSelector
- DifficultySelector
- Count input (1-10, default 5)
- Generate button
- Handles form submission via useGenerateQuestions mutation

**Acceptance criteria:**
- [ ] All components created in questions/ directory
- [ ] Proper TypeScript props
- [ ] Uses existing UI patterns (buttons, cards, badges from existing components)
- [ ] Form validation (material required, topic required, at least one type)
