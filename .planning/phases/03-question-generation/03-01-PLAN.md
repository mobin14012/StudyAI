# Plan 03-01: Backend Question Model + AI Generator

## Goal

Create the Question Mongoose model, AI question generator service, validation utilities, and caching logic. This plan establishes all the core backend infrastructure for question generation without exposing API routes yet.

## Prerequisites

- Phase 2 complete (Material model, OpenAI client, AI services exist)
- MongoDB running
- OpenAI API key configured

## Tasks

### Task 1: Create Question Model

**File:** `server/src/models/Question.ts`

Create the Mongoose model with:
- Fields: userId, materialId, type, difficulty, text, options, correctAnswer, explanation, topic, sourceExcerpt, cacheKey
- Proper TypeScript types (QuestionType, DifficultyLevel, IQuestion)
- Indexes for common queries and cache lookups
- Validation: MCQ must have exactly 4 options

**Acceptance criteria:**
- [ ] IQuestion interface exported
- [ ] QuestionType and DifficultyLevel types exported
- [ ] Schema validates MCQ options correctly
- [ ] All required indexes defined

### Task 2: Create Cache Key Utility

**File:** `server/src/utils/cache-key.ts`

Create utility function for generating cache keys:
- `generateCacheKey(materialId, topic, difficulty, type)` returns a 32-char SHA-256 hash
- Export function for use in service and model

**Acceptance criteria:**
- [ ] Deterministic: same inputs always produce same hash
- [ ] Case-insensitive topic handling

### Task 3: Create Question Generator AI Service

**File:** `server/src/services/ai/question-generator.ts`

Create the AI service that generates questions:
- System prompt with grounding rules and output format
- Difficulty-specific guidelines (easy/medium/hard)
- User prompt builder with source text, topic, difficulty, types, count
- Response parsing and validation
- Integration with existing `chatCompletion` from `openai-client.ts`

**Acceptance criteria:**
- [ ] `generateQuestions()` function exported
- [ ] Uses `chatCompletion` with jsonMode: true
- [ ] Includes sourceExcerpt in response schema
- [ ] Handles batch generation (multiple types/questions)

### Task 4: Create Question Validation Utilities

**File:** `server/src/services/ai/question-validator.ts`

Create validation utilities:
- Zod schema for generated question structure
- `validateGeneratedQuestions()` - validates array of questions
- `validateSourceGrounding()` - verifies sourceExcerpt exists in source text
- MCQ-specific validation (4 options, answer in options)
- True/False-specific validation (answer is "True" or "False")

**Acceptance criteria:**
- [ ] Returns ValidationResult with valid questions and filtered count
- [ ] Logs warnings for filtered questions
- [ ] 80% word match threshold for source grounding

### Task 5: Create Question Validation Schemas

**File:** `server/src/schemas/question.schemas.ts`

Create Zod schemas for API request validation:
- `generateQuestionsSchema` - materialId, topic, difficulty?, types[], count?
- `questionListQuerySchema` - filters and pagination
- `questionIdParamSchema` - single ID validation
- `batchDeleteSchema` - array of IDs

**Acceptance criteria:**
- [ ] All schemas exported
- [ ] Type inference exports for each schema
- [ ] Proper defaults (difficulty from user level, count=5, types=["mcq"])

### Task 6: Create Question Service

**File:** `server/src/services/question.service.ts`

Create the orchestration service:
- `generateOrGetCachedQuestions()` - main entry point with caching logic
- `getMaterialForGeneration()` - fetch and validate material
- `getDefaultDifficulty()` - returns difficulty based on user level
- `saveQuestions()` - bulk insert validated questions
- Retry logic for failed AI generation

**Acceptance criteria:**
- [ ] Cache lookup before AI generation
- [ ] Only generates missing question types
- [ ] Applies user-level default difficulty (QGEN-04)
- [ ] Returns { questions, generated, cached, totalReturned }

## Implementation Order

1. Task 2 (cache-key.ts) - no dependencies
2. Task 1 (Question.ts) - uses cache key
3. Task 4 (question-validator.ts) - uses Zod
4. Task 3 (question-generator.ts) - uses openai-client
5. Task 5 (question.schemas.ts) - standalone
6. Task 6 (question.service.ts) - uses all above
