# Plan 07-02: Mobile Responsiveness Audit

## Objective

Audit all pages for mobile responsiveness on common phone screen sizes (375px, 390px, 414px width). Fix layout issues, ensure touch targets are at least 44px, and verify charts/tables are usable on mobile.

## Requirements Addressed

- **UIUX-03**: All pages are mobile responsive (usable on phone screens)

## Context

### Files to Read First
- `client/src/components/layout/Sidebar.tsx` - Navigation component
- `client/src/components/layout/Layout.tsx` - Main layout wrapper
- `client/src/pages/AnalyticsPage.tsx` - Charts/data visualization
- `client/src/pages/PracticePage.tsx` - Practice flow
- `client/src/components/practice/*.tsx` - Practice components

### Dependencies
- Plan 07-01 complete (UI polish applied)
- Tailwind CSS responsive utilities available
- shadcn/ui responsive dialog components

### Mobile Viewport Targets
- iPhone SE: 375px width
- iPhone 14: 390px width
- iPhone 14 Plus: 428px width
- Common Android: 360-412px width

### Known Responsive Patterns in Codebase
- Container: `container mx-auto px-4`
- Grid: `grid-cols-1 md:grid-cols-2`
- Hide on mobile: `hidden md:block`
- Show on mobile: `md:hidden`

## Tasks

### Task 1: Audit and Fix Sidebar Navigation

**File:** `client/src/components/layout/Sidebar.tsx`
**File:** `client/src/components/layout/Layout.tsx`

**Current State:** Read the files to understand current implementation.

**Required Behavior:**
1. Sidebar should be hidden on mobile (< 768px)
2. Mobile hamburger menu should show/hide sidebar as a slide-over
3. Use shadcn Sheet component for mobile drawer

**Implementation Pattern:**
```tsx
// Layout.tsx - Mobile drawer + desktop sidebar
import { Sheet, SheetContent, SheetTrigger } from "@/components/ui/sheet";
import { Button } from "@/components/ui/button";
import { Menu } from "lucide-react";

// Desktop: sidebar visible
// Mobile: hamburger button triggers Sheet drawer

<>
  {/* Mobile Header with Hamburger */}
  <div className="md:hidden fixed top-0 left-0 right-0 h-14 border-b bg-background z-50 flex items-center px-4">
    <Sheet>
      <SheetTrigger asChild>
        <Button variant="ghost" size="icon">
          <Menu className="h-5 w-5" />
        </Button>
      </SheetTrigger>
      <SheetContent side="left" className="w-64 p-0">
        <Sidebar />
      </SheetContent>
    </Sheet>
    <span className="ml-4 font-semibold">StudyAI</span>
  </div>

  {/* Desktop Sidebar */}
  <aside className="hidden md:block fixed left-0 top-0 h-full w-64 border-r bg-background">
    <Sidebar />
  </aside>

  {/* Main Content */}
  <main className="pt-14 md:pt-0 md:ml-64">
    {children}
  </main>
</>
```

### Task 2: Audit Practice Page Components

**Files to check:**
- `client/src/components/practice/PracticeSetup.tsx`
- `client/src/components/practice/QuestionDisplay.tsx`
- `client/src/components/practice/McqOptions.tsx`
- `client/src/components/practice/TrueFalseButtons.tsx`
- `client/src/components/practice/ShortAnswerInput.tsx`
- `client/src/components/practice/FeedbackDisplay.tsx`

**Checklist for each:**
1. Touch targets (buttons) are at least 44px height
2. Text is readable (min 14px for body text)
3. Cards have appropriate padding (`p-4` or more on mobile)
4. No horizontal overflow on small screens
5. Forms are full-width on mobile

**Common Fixes:**
```tsx
// Ensure buttons have adequate touch targets
<Button size="lg" className="h-12">  {/* 48px height */}

// Full-width buttons on mobile
<Button className="w-full md:w-auto">

// Responsive text sizing
<h1 className="text-xl md:text-2xl lg:text-3xl">

// Responsive padding
<div className="p-4 md:p-6">
```

### Task 4: Audit Form Pages

**Files:**
- `client/src/pages/LoginPage.tsx`
- `client/src/pages/RegisterPage.tsx`
- `client/src/pages/UploadPage.tsx`
- `client/src/pages/ProfilePage.tsx`

**Checklist:**
1. Form containers are max-width constrained (`max-w-md mx-auto`)
2. Input fields are full-width
3. Labels are visible and associated
4. Submit buttons are full-width on mobile
5. Error messages don't cause layout shift

**Pattern:**
```tsx

      <Input id="email" type="email" className="w-full" />
    </div>
    <Button type="submit" className="w-full">
      Submit
    </Button>
  </form>
</div>
```
