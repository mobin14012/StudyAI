# Plan 07-03: Production Deployment

## Objective

Deploy the StudyAI application to production: frontend on Vercel, backend on Render, database on MongoDB Atlas. Verify the complete end-to-end flow works in production.

## Requirements Addressed

This plan addresses the deployment success criteria rather than specific feature requirements:
- App is deployed and accessible
- End-to-end flow works: register → upload → generate → practice → see analytics

### Task 1: Create Health Check Endpoint

**File:** `server/src/routes/health.routes.ts` (CREATE)

```typescript
import { Router } from "express";
import mongoose from "mongoose";

const router = Router();

// GET /api/health — Health check for monitoring
router.get("/", (_req, res) => {
  const dbState = mongoose.connection.readyState;
  const dbStatus = dbState === 1 ? "connected" : "disconnected";
  
  res.json({
    status: "ok",
    timestamp: new Date().toISOString(),
    database: dbStatus,
  });
});

export default router;
```

**File:** `server/src/app.ts` (MODIFY)

Add health route registration (no auth required):


## Deployment Steps

### Step 1: Set Up MongoDB Atlas

1. Go to https://cloud.mongodb.com
2. Create free cluster (M0 tier)
3. Create database user with password
4. Add IP address `0.0.0.0/0` to whitelist (allows Render's dynamic IPs)
5. Get connection string (replace `<password>` with actual password)

### Step 2: Deploy Backend to Render

1. Go to https://render.com
2. Connect GitHub repository
3. Create new "Web Service"
4. Select the repository
5. Configure:
   - **Name:** studyai-api
   - **Root Directory:** `server`
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm start`
6. Add environment variables from `.env.production.example`
7. Deploy and wait for health check to pass
8. Note the URL (e.g., `https://studyai-api.onrender.com`)

### Step 3: Deploy Frontend to Vercel

1. Go to https://vercel.com
2. Import GitHub repository
3. Configure:
   - **Framework Preset:** Vite
   - **Root Directory:** `client`
   - **Build Command:** `npm run build`
   - **Output Directory:** `dist`
4. Add environment variable:
   - `VITE_API_URL` = Render backend URL (from Step 2)
5. Deploy
6. Note the URL (e.g., `https://studyai.vercel.app`)

### Step 4: Update Backend CLIENT_URL

1. Go back to Render dashboard
2. Update `CLIENT_URL` environment variable to Vercel URL
3. Redeploy backend

## Troubleshooting

### Common Issues

**CORS Errors:**
- Verify `CLIENT_URL` in Render matches exactly (no trailing slash)
- Check browser DevTools Network tab for blocked requests

**Cookie Not Set:**
- Verify `secure: true` only in production
- Verify `sameSite: "lax"` setting
- Both frontend and backend must use HTTPS

**Database Connection Failed:**
- Verify MongoDB Atlas whitelist includes `0.0.0.0/0`
- Verify connection string credentials are correct
- Check Render logs for connection errors

**Build Failures:**
- Check Render/Vercel build logs
- Verify all dependencies are in `dependencies` (not `devDependencies` for runtime)
- Ensure TypeScript builds without errors locally first
